Interface IConnectionAcceptor

An IConnectionAcceptor represents an IConnectionManager running on a server, and as a server, it is responsible for accepting a connection request initiated by a IConnectionInitiator.

Remarks

Before a connection can be accepted, the IConnectionAcceptor must be started using the Start() method. Calling this method allocates any necessary resources and transitions the IConnectionAcceptor to the running state. The IConnectionAcceptor will then accept new connections, which are represented by IConnection object. The IConnectionAcceptor maintains references to accepted IConnection objects until they are closed or the IConnectionAcceptor is Shutdown() or Stop() terminated. Terminating a IConnectionAcceptor also closes all accepted IConnection objects.

All IConnectionAcceptor implementations must be fully thread-safe.
