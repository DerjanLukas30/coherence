Interface IConnection

An IConnection represents a bi-directional communication channel for exchanging IMessage objects between two endpoints.

Namespace: Tangosol.Net.Messaging
Assembly: Coherence.Core.dll
Syntax
public interface IConnection
Remarks

An IConnection is created by an IConnectionInitiator running on a client. Before an IConnection can be established, the server must initialize and start an IConnectionAcceptor. Creating the IConnection results in a handshake between the client and the server (including IProtocol negotiation); if the handshake is successful, the IConnection will be transitioned to the open state.

An IConnection itself is not used to send and receive IMessage objects. Rather, one or more IChannel objects may be created from the IConnection and used to exchange IMessage objects of a known IProtocol with a peer. An IChannel may be OpenChannel(IProtocol, String, IReceiver, IPrincipal) opened to a named IReceiver registered by the peer. Alternatively, an "anonymous" IChannel (a back-Channel) may be CreateChannel(IProtocol, IReceiver) created by one peer and returned to the other, where it must be AcceptChannel(Uri, IReceiver, IPrincipal) accepted before it can be used.

Once finished with the IConnection, an application should release all resources held by the IConnection by calling the Close() method. Closing an IConnection also closes all IChannel objects created by the IConnection and renders the IConnection unusable. Attempting to use a closed IConnection or any IChannel created by a closed IConnection may result in an exception.

All IConnection implementations must be fully thread-safe.

Type Description
System.Uri

A Uri that represents the new IChannel object.
Remarks

This method is particularly useful for building a Response Message to send back a new IChannel that can be used by the peer. In practice, this means that when a call to a stub is made, it can easily return a new stub that has its own IChannel; for example, a stub representing one service can return a stub representing a different service.

The new IChannel cannot be used until the returned Uri is AcceptChannel(Uri, IReceiver, IPrincipal) accepted by the peer.

OpenChannel(IProtocol, String, IReceiver, IPrincipal)

Create an IChannel using a specific IProtocol through this IConnection to a named IReceiver on the other end of the IConnection, optionally providing an IPrincipal to indicate the identity that will be utilizing the IChannel, and optionally providing an IReceiver that will process unsolicited IMessage objects on this end of the IChannel.

Declaration
IChannel OpenChannel(IProtocol protocol, string name, IReceiver receiver, IPrincipal principal)
Parameters
Type Name Description
IProtocol protocol

The IProtocol that will be used to communicate through the IChannel; the IProtocol is used to verify that the IReceiver on the peer with the specified name is capable of communicating using that IProtocol.
System.String name

The name that the IReceiver was registered with, on the other end of this IConnection; an IReceiver with the specified name must have been registered with the peer's IConnectionManager prior to calling this method (see RegisterReceiver(IReceiver)).
IReceiver receiver

An optional IReceiver to associate with this IChannel that will process any unsolicited IMessage objects sent back through the IChannel by the peer.
IPrincipal principal

An optional IPrincipal to associate with this IChannel; if specified, any operation performed upon receipt of an IMessage sent using the returned IChannel will be done on behalf of the specified IPrincipal.
Returns
Type Description
IChannel

A new IChannel object.
Remarks

Conceptually, this is how an IChannel is established to an existing named "service" (e.g. an IReceiver) on the peer; note that either peer can register named services and either peer can use this method to find a named service on its peer.
