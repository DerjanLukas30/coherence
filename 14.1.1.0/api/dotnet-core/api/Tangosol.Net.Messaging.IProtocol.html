Interface IProtocol

An IProtocol is a binding between a unique name, version information, and a set of IMessage types.

Namespace: Tangosol.Net.Messaging
Assembly: Coherence.Core.dll
Syntax
public interface IProtocol
Remarks

It is used to describe the types of IMessage objects (the "dialect", so to speak) that can be exchanged between two endpoints through an IChannel via a IConnection.

Before an IConnection can be created or accepted, one or more IProtocol instances must be registered with the client and server-side IConnectionManager. During IConnection establishment, the client's IConnectionInitiator sends information about each registered IProtocol. A compatable set of IProtocol objects (or superset) must be registered with server's IConnectionManager in order for the the IConnection to be accepted.
