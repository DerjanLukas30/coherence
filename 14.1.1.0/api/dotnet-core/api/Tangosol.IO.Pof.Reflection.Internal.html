Classes

AnnotationVisitor<T, TB>

An AnnotationVisitor is a IVisitor<B> implementation that injects information obtained by inspecting a provided type into a TypeMetadataBuilder<T>. The modified builder will then realize a ITypeMetadata<T> instance with this injected information.

InvocationStrategies

InvocationStrategies contains two IInvocationStrategy<T> implementations that abstract the underlying mechanisms to retrieve and set a property's value.

InvocationStrategies.FieldInvcationStrategy<T>

A FieldInvocationStrategy uses a System.Reflection.FieldInfo to dynamically invoke gets and sets on the field.

InvocationStrategies.MethodInvocationStrategy<T>

A MethodInvocationStrategy uses System.Reflection.MethodInfos to dynamically invoke getter and setter methods to retrieve and set property values.

InvocationStrategies.PropertyInvcationStrategy<T>

A PropertyInvcationStrategy uses a System.Reflection.PropertyInfo to dynamically invoke gets and sets on the property.

NameManglers

NameManglers contain singleton access to both a NameManglers.FieldMangler and NameManglers.MethodMangler. NameManglers provide the ability to derive the same name of a property regardless of their access or inspection methodology.

NameManglers.FieldMangler

A INameMangler implementation that is aware of field naming conventions and is able to convert from a field name to a generic name.

NameManglers.MethodMangler

A INameMangler implementation that is aware of method naming conventions and is able to convert from a method name to a generic name.

NameManglers.PropertyMangler

A INameMangler implementation that is aware of property naming conventions and is able to convert from a method name to a generic name.

TypeMetadata<T>

A ITypeMetadata<T> implementation coupled to the .NET type metadata definition language: System.Type, System.Reflection.PropertyInfo, System.Reflection.FieldInfo, and System.Reflection.MethodInfo.

TypeMetadata<T>.TypeAttribute

An IAttributeMetadata<T> implementation acting as a container for attribute inspection and invocation.

TypeMetadata<T>.TypeKey

A TypeKey contains information to uniquely identify this
type instance. Specifically unique identification is a product of

typeId + version + type-hash
.

TypeMetadataBuilder<T>

TypeMetadataBuilder provides a simple mechanism to instantiate and inject state into a ITypeMetadata<T> instance. Parsers that read a source will use this builder to derive a ITypeMetadata<T> destination.

TypeMetadataBuilder<T>.ClassAttributeBuilder

The ClassAttributeBuilder provide the ability to build a IAttributeMetadata<T> implementation.

Interfaces

IAttributeMetadata<T>

IAttributeMetadata represents all appropriate information relating to an attribute within a type. This contract has similar forms in all supported languages providing a language agnostic mechanism to describe elements within a structure and an invocation mechanism for setting or retrieving the value for an attribute.

IInvocationStrategy<T>

An IInvocationStrategy provides an abstraction of the underlying mechanisms used to retrieve and set a property's value.

INameMangler

INameMangler implementations provide the ability to transform a string to the string convention employed by the mangler implementation. An example of this would be to convert a non-camel case string to a camel case string.

IRecipient<B>

A recipient informs a visitor of it's willingness to be visited.

ITypeKey

A type key embodies contributors to the uniqueness representing an ITypeMetadata instance. This is the sum of typeId, versionId and a hash.

ITypeMetadata<T>

ITypeMetadata represents the definition of a type including uniqueness ITypeKey and all structural properties. This definition is used to uniformly define types and their internal structures. Uniformity in this context is in relation to the supported languages.

IVisitor<B>

Visitor pattern description. This pattern implementation is targeted at builders that require System.Type information.
