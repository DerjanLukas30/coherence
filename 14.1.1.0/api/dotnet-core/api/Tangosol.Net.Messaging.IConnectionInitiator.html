Interface IConnectionInitiator

An IConnectionInitiator represents an IConnectionManager running on a client, and as a client, it is responsible for initiating the connection process.

public interface IConnectionInitiator : IConnectionManager, IService, IControllable
Remarks

Before a connection can be established, the IConnectionInitiator must be started using the Start() method. Calling this method allocates any necessary resources and transitions the IConnectionInitiator to the running state. Additionally, the server endpoint must have initialized and started a IConnectionAcceptor. The IConnectionInitiator can then be used to establish a connection to the server's IConnectionAcceptor, which is represented by a single IConnection object, obtained by calling EnsureConnection(). The IConnectionInitiator maintains a reference to the IConnection object until it is closed or the IConnectionInitiator is Shutdown() or Stop() terminated.

All IConnectionInitiator implementations must be fully thread-safe.

Create a new or return the existing IConnection object.

Declaration
IConnection EnsureConnection()
Returns
Type Description
IConnection

An IConnection object representing a client's connection to a server.
Remarks

An IConnection object has a one-way state transition from open to closed; this method will always return an open IConnection object. If the previously existing IConnection object has transitioned to a closed state, this method will return a new IConnectin object in the open state.
