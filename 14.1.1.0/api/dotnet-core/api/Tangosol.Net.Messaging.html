Interfaces

IChannel

An IChannel is a communication construct that allows one or more threads to send and receive IMessage objects via a IConnection.

ICodec

An ICodec converts an IMessage object to and from a binary representation.

IConnection

An IConnection represents a bi-directional communication channel for exchanging IMessage objects between two endpoints.

IConnectionAcceptor

An IConnectionAcceptor represents an IConnectionManager running on a server, and as a server, it is responsible for accepting a connection request initiated by a IConnectionInitiator.

IConnectionInitiator

An IConnectionInitiator represents an IConnectionManager running on a client, and as a client, it is responsible for initiating the connection process.

IConnectionManager

The IConnectionManager is the base SPI (Service Provider Interface) for both IConnectionAcceptor and IConnectionInitiator implementations.

IMessage

IMessage is the root interface for all message objects sent by peer endpoints through an IChannel.

IMessageFactory

An IMessageFactory is a factory for IMessage objects.

IProtocol

An IProtocol is a binding between a unique name, version information, and a set of IMessage types.

IReceiver

An IReceiver processes unsolicited IMessage objects sent via any number of IChannel objects.

IRequest

IRequest is the root interface for all request messages sent by peer endpoints through an IChannel.

IResponse

IResponse is the root interface for all response messages sent by peer endpoints through an IChannel.

IRunnable

Defines method Run() that executes action specific to the object in which it is implemented.

IStatus

An IStatus represents an asynchronous IRequest sent to a peer.
