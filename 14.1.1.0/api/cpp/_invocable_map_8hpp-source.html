00354         /**
00355         * Invoke the passed EntryProcessor against the set of entries that
00356         * are selected by the given Filter, returning the result of the
00357         * invocation for each.
00358         *
00359         * Unless specified otherwise, InvocableMap implementations will
00360         * perform this operation in two steps: (1) use the filter to retrieve

00363         * the result of the specified filter evaluation, since the filtering
00364         * and processing could be performed in parallel on different threads.
00365         * If this assumption does not hold, the processor logic has to be
00366         * idempotent, or at least re-evaluate the filter. This could be
00367         * easily accomplished by wrapping the processor with a
00368         * ConditionalProcessor.
00369         *
00370         * @param vFilter  a Filter that results in the set of keys to be
00371         *                 processed
00372         * @param hAgent   the EntryProcessor to use to process the specified
00373         *                 keys
00374         *
00375         * @return a Map containing the results of invoking the EntryProcessor
00376         *         against the keys that are selected by the given Filter
00377         */
